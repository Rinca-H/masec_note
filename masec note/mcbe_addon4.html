アドオン作成 - ブロックの作り方

目次は右上の「≡」から！

　オリジナルのブロックの作り方・追加方法を紹介します。 オリジナルのものを「追加する」ことになるので、ある程度アドオンに慣れていないとつまずくかもしれません。
　複雑なものほど急に難しくなるので注意してください。

基本的な流れ

　もともとマイクラになかったものを追加するので、ブロック本体に加えてテクスチャも追加する必要があります。 つまり、ビヘイビアとリソース両方が必要になります。
　全体の作業はこんな感じです：

ビヘイビア
・blocksフォルダに新しくJSONを追加する
(・ルートテーブルのJSONを追加する)
リソース
・blocks.jsonを編集・作成する
・texturesフォルダのterrain_texture.jsonを編集・作成する
・ブロックのテクスチャを追加する
(・ブロックのジオメトリを追加する)

ブロック本体を定義

　まず、ビヘイビアパック内にblocksフォルダを用意します。

━ ┳ manifest.json
　 ┣ blocks
　 ┗ pack_icon.png

　そのフォルダのなかにブロック本体を定義するJSONを、追加するブロックの数だけ作って入れることになります。

テクスチャを用意

　自分で場所を指定することができますが、オススメは「(リソースパック)/textures/blocks」の中です。

━ ┳ manifest.json
　 ┣ pack_icon.png
　 ┗ textures
　 　 ┗blocks
　 　 　 ┗〇〇.png

テクスチャと本体を紐づけ①

「(リソースパック)/textures」の中に「terrain_texture.json」を用意します。
　元からあれば編集、なければ新しく作ります。
　詳しくはこちら → terrain_texture.jsonについて

テクスチャと本体を紐づけ②

　リソースパックの中(manifest.jsonと同じ層)に「blocks.json」を用意します。
　こちらも同様、元からあれば編集、なければ新しく作ります。
　詳しくはこちら → blocks.jsonについて

ブロック本体(基礎)

　これがまだ何も指定していない状態のブロックです。ゼロから作る場合はこれをコピペするといいでしょう。

Dat
../blocks/my_block.json
{
  "format_version": "1.16.100",
  "minecraft:block": {
    "description": {
      "identifier": "test:my_block"
    },
    "components": {
      // コンポーネント
    }
  }
}

主な構成

{ "format_version": "1.16.100", "minecraft:block": { "description": { "identifier": "test:my_block" },
"components": { // コンポーネント }
} }

　大きく関わってくるのがこの赤い枠で囲った部分で、もっと言えばその中の{ }内にコンポーネントを書き足していきます。

IDを決める

{ "format_version": "1.16.100", "minecraft:block": { "description": { "identifier": "test:my_block" },
"components": { // コンポーネント }
} }

　IDを決め忘れないでください。

？
identifier(ID)の決め方
　基本的には「(アドオンid):(本id)」といった感じ。
　アドオンidはアドオン全体で統一する。苗字のようなもの。ネームスペースとかプリフィックスとか呼ばれることがある。
　本idは文字通り本体のid。みんな違うものをつけましょう。
例)　test:my_content
--注意--
・どちらも先頭を数字にしない。
・数字かアルファベットのみを使う。
・アドオンidに「minecraft」は使ってはならない。
・アドオンidは、ほかのアドオンとの競合を考えてオリジナル性の高いものにする。

コンポーネントを追加する

{ "format_version": "1.16.100", "minecraft:block": { "description": { "identifier": "test:my_block" },
"components": { "minecraft:block_light_emission": 1.0 }
} }

　リファレンスから選んでコンポーネントを追加します。 何を選んだらいいのかわからない場合は後で例をいくつか載せるので参考にしてください。

？
コンポーネントの選び方

　Blocks.htmlというリファレンスの「Block Components」「Block Trigger Components」の章に載っているものはすべてコンポーネントです。
　そこには表があって、

名前型デフォルト値説明
minecraft:block_light_absorptionInteger0(略)

　例えばこのようになっていたら「"minecraft:block_light_absorption": (整数)」がコンポーネントの形になります。

リファレンス(アドオンを作るひとへ)
最も簡単な例

　JSONがシンプルになるように性質を選んできています。

Dat
../blocks/my_block.json
{
  "format_version": "1.16.100",
  "minecraft:block": {
    "description": {
      "identifier": "test:simple_block"
    },
    "components": {
      "minecraft:block_light_emission": 1.0,
      "minecraft:creative_category": {
        "category": "construction",
        "group": "itemGroup.name.slab"
      },
      "minecraft:destroy_time": 1.0,
      "minecraft:explosion_resistance": 5.0,
      "minecraft:loot": "loot_tables/blocks/my_block.loot_table.json",
      "minecraft:map_color": "#64ec86"
    }
  }
}

　このお手本ブロックは以下のような性質を持ちます↓

・明るさレベル最大で光る
・クリエイティブインベントリでは「建造物」カテゴリにある
・壊すのにかかる時間は1.0秒
・爆発耐性は5.0
・ルートテーブルは「loot_tables/blocks/my_block.json」を参照
・地図での表示色は #64ec86

　はじめはこれをコピペして自分で値を変えて試すのがいいかもしれません。
　それぞれの性質についてもっと詳しく説明します。

minecraft:block_light_emission

　光源となってどれぐらい光るかか0.0~1.0の小数で指定します。
　0.0で光らない、1.0で明るさレベル15。範囲を超えるとエラー。

minecraft:creative_category

　クラフトインベントリでどこに表示するか。
　categoryは"Construction"(構築), "Equipment"(所持品), "Items"(アイテム), "Nature"(性質)から選び、クラフトインベントリのタブを指定します。
　groupは木材やハーフブロックのようにグループとして折りたたまれているマスです。

minecraft:destroy_time

　このブロックの破壊にかかる秒数を小数で指定します。破壊不可にはできません。
　この値は爆発耐性に影響します

minecraft:explosion_resistance

　このブロックの爆発耐性。大きいほど強い爆発に耐えます。

minecraft:loot

　ルートテーブルのJSONの場所(ファイルパス)を指定します。
　オススメはloot_tables/blocks/〇〇.jsonです。

minecraft:map_color

　マップでの表示色を16進カラーコードで指定します。16進カラーコードについてはググればすぐ出てきます。
