アドオン作成 - ブロックの作り方

　オリジナルのブロックの作り方・追加方法を紹介します。 オリジナルのものを「追加する」ことになるので、ある程度アドオンに慣れていないとつまずくかもしれません。
　複雑なものほど急に難しくなるので注意してください。

基本的な流れ

　もともとマイクラになかったものを追加するので、ブロック本体に加えてテクスチャも追加する必要があります。 つまり、ビヘイビアとリソース両方が必要になります。
　まず、ビヘイビアパック内にblocksフォルダを追加します。

━ ┳ manifest.json
　 ┣ blocks
　 ┗ pack_icon.png

　このフォルダの中にJSONファイルを追加してブロックを追加します。
　以降の作業はこんな感じです：

ビヘイビア
・blocksフォルダに新しくJSONを追加する
・ルートテーブルのJSONを追加する(任意)
リソース
・blocks.jsonを編集・作成する
・texturesフォルダのterrain_texture.jsonを編集・作成する
・ブロックのテクスチャを追加する
・ブロックのジオメトリを追加する(任意)

　上から順に説明していきます。ほとんどどれも抜かしてはいけない手順なので注意してください。

ブロック本体(レベル1)

　まずは基本的なブロックです。簡単な性質だけを持ったシンプルなブロックを追加して作業行程を身につけましょう。

Dat
../blocks/my_block.json
{
  "format_version": "1.16.100",
  "minecraft:block": {
    "description": {
      "identifier": "test:my_block"
    },
    "components": {
      "minecraft:block_light_emission": 1.0,
      "minecraft:creative_category": {
        "category": "construction",
        "group": "itemGroup.name.slab"
      },
      "minecraft:destroy_time": 1.0,
      "minecraft:explosion_resistance": 5.0,
      "minecraft:loot": "loot_tables/blocks/my_block.loot_table.json",
      "minecraft:map_color": "#64ec86",
      "tag:my_block_tag": {}
    }
  }
}

　概形はこんな感じです。シンプルなJSONになるように性質を選んできています。
　このお手本ブロックは↓のような性質を持ちます。

・明るさレベル最大で光る
・クリエイティブインベントリでは「建造物」カテゴリにある
・壊すのにかかる時間は1.0秒
・爆発耐性は5.0
・ルートテーブルは「loot_tables/blocks/my_block.json」を参照
・地図での表示色は #64ec86
・「my_block_tag」というブロックタグがついている

　はじめはこれをコピペして値を自分で変えて試すのがいいかもしれません。 4行目のidentifierは自分のものに変えておきましょう。

identifier(id・ID)の決め方
　基本的には「(アドオンid):(本id)」といった感じ。
　アドオンidはアドオン全体で統一する。苗字のようなもの。ネームスペースとかプリフィックスとか呼ばれることがある。
　本idは文字通り本体のid。みんな違うものをつけましょう。
--注意--
・どちらも先頭を数字にしない。
・数字かアルファベットのみを使う。
・アドオンidに「minecraft」は使ってはならない。
・アドオンidは、ほかのアドオンとの競合を考えてオリジナル性の高いものにする。

　それぞれの性質についてもっと詳しく説明します。

記号の説明
記号対応する型
Strストリング型
Int整数型
Float小数型
Boolブーリアン型
Objオブジェクト型
Vec-n配列型

minecraft:block_light_emission　[Float(0.0~1.0)]

　光源となってどれぐらい光るかか0.0~1.0の値で指定します。0.0で光らない、1.0で明るさレベル15。範囲を超えるとエラー。

minecraft:creative_category　[Obj]

　クリエイティブインベントリでどこに表示するか。

minecraft:destroy_time　[Float(≥0.0)]

　このブロックの破壊にかかる秒数。破壊不可にはできません。

minecraft:explosion_resistance　[Float(≥0.0)]

　このブロックの爆発耐性。大きいほど強い爆発に耐えます。

minecraft:loot　[Str(ルートテーブル)]

　ルートテーブルのJSONの場所(ファイルパス)を指定します。
　オススメはloot_tables/blocks/〇〇.jsonです。

minecraft:map_color　[Str(16進カラーコード)]

　マップでの表示色を16進カラーコードで指定します。16進カラーコードについてはググればすぐ出てきます。

　レベル2以降は飛ばして次の手順に移ります。レベル2以降についてはこのページの下のほうに書きます。

blocks.jsonを編集・作成

●新しく作る場合

　リソースパックのmanifest.jsonと同じ層にblocks.jsonがない場合は新しく作ります。

━ ┳ blocks.json
　 ┣ manifest.json
　 ┗ pack_icon.png
Res
../blocks.json
{
  "format_version": [ 1, 1, 0 ],
  "test:my_block": {
    "textures": "my_block",
    "sound": "stone"
  }
}

　中身はこれをコピペでいいです。

●既にある場合

　中身を書き加える場合です。 ちなみに、バニラの情報が書いてあるならいらないので全部消して「新しく作る場合」を参照してください。

Res
../blocks.json
{
  "format_version": [ 1, 1, 0 ],
  "〇〇": {
    "textures": "〇〇",
    "sound": "〇〇"
  },
  　⫶
  "〇〇": {
    "textures": "〇〇",
    "sound": "〇〇"
  }
}

　このように既に書いてあるとします。
　基本はどこに追加してもいいですが、分かりやすく一番上へ書くことにします。
