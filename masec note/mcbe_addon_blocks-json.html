アドオン作成 - blocks.jsonについて

目次は右上の「≡」から！

　このファイルはブロックのテクスチャに関するファイルで、リソースパックに一つだけあります。

━ ┳ blocks.json
　 ┣ manifest.json
　 ┗ pack_icon.png
機能

　ブロックにどんなテクスチャを貼るかを決定するための中間行程です。ブロックのサウンドもここで決めます。
　主な役割として、ブロック本体とTerrainIDを結びつけます。

ブロック本体
　↓　block.jsonで紐づけ
TerrainID
　↓　terrain_texture.jsonで紐づけ

TerrainID(terrain_textures.jsonについて)
ファイル内容

　本来は、テクスチャを紐付けしたいブロックすべてについて書きますが、はじめは例として一つのブロックの場合を示します。

Res
blocks.json
{
  "format_version": [ 1, 1, 0 ],
  "minecraft:cobblestone": {
    "sound": "stone",
    "textures": "cobblestone"
  }
}

　概形はこんな感じです。
　自分で新しく作る場合はこれ↑をコピペするといいでしょう。

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "sound": "stone", "textures": "cobblestone" }
}

　この赤線で囲った部分が一つのカタマリです。これを適当に「ユニット」と呼ぶことにします。
(このサイト独自の呼び方です)

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "sound": "stone", "textures": "cobblestone" }
}

　ここはブロック本体のIDです。「minecraft:」なら省略できます。

sound

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "sound": "stone", "textures": "cobblestone" }
}

　ここでブロックに関わる効果音を指定します。

textures

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "sound": "stone", "textures": "cobblestone" }
}

　ここではTerrainIDを指定します。

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "sound": "stone",
"textures": { "up": "cobblestone", "down": "cobblestone", "north": "cobblestone", "south": "cobblestone", "east": "cobblestone", "west": "cobblestone" }
}
}

　このように面ごとで分けて指定することもできます。trueにする面だけ書くことをオススメします。
　ちなみに："up"→上面, "down"→下面, "north"→北面, "north"→南面, "east"→東面, "west"→西面

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "sound": "stone",
"textures": { "up": "cobblestone", "down": "cobblestone", "side": "cobblestone" }
}
}

　さらに、東西南北の面すべてをまとめて指定する場合は"side"でまとめられます。

？
TerrainID(terrain_textures.jsonについて)

指定するブロックを増やす

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "sound": "stone", "textures": "cobblestone" },
"minecraft:dirt": { "sound": "gravel" "textures": "dirt" }
}

　複数のブロックについて記述する場合は、ユニット自体を増やします。
　ユニットとユニットをつなぐときのカンマ( , )は忘れるとエラーになります。

ブロックにテクスチャがつくまで

あわせて読んでおきたい(terrain_textures.jsonについて)

　普通に6面同じテクスチャなら単純です。textures/blocksにある「img.png」を「test:my_block」というidのブロックに貼るとします。
　そして、TerrainIDも自分で決められるのでyamaimoとします。すると、

Res
textures/terrain_textures.json
{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": {
"yamaimo": { "textures": "textures/blocks/img" }
} }

Res
blocks.json
{ "format_version": [ 1, 1, 0 ],
"test:my_block": { "sound": "stone", "textures": "yamaimo" }
}

　このようにTerrainIDを介してテクスチャとブロック本体が結びつきます。
　一方、面で異なる場合は慣れていないと一見複雑に感じます。 例として、上面、側面、下面で異なるテクスチャ「img_up.png」「img_side.png」「img_down.png」であるとしましょう。この場合、

Res
textures/terrain_textures.json
{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": {
"yamaimo_up": { "textures": "textures/blocks/img_up" },
"yamaimo_side": { "textures": "textures/blocks/img_side" },
"yamaimo_down": { "textures": "textures/blocks/img_down" }
} }
※ TerrainIDは自由に決められます
Res
blocks.json
{ "format_version": [ 1, 1, 0 ],
"test:my_block": { "sound": "stone", "textures": { "up": "yamaimo_up", "side": "yamaimo_side", "down": "yamaimo_down" } }
}

　というようにTerrainIDを分岐させて指定します。

発展的な内容

　ユニットの中身の項目にはほかにもまだ存在します。 

{
  "format_version": [ 1, 1, 0 ],
  "minecraft:cobblestone": {
    "brightness_gamma": 0,
    "carried_textures": "cobblestone",
    "isotropic": false,
    "sound": "stone",
    "textures": "cobblestone"
  }
}

　全項目書いてみましたが、いらない項目は一行まるごと消してください。

brightness_gamma

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "brightness_gamma": 0, "carried_textures": "cobblestone", "isotropic": false, "sound": "stone", "textures": "cobblestone" }
}

　このブロックの下にあるブロックの見た目の明るさを指定します。正直必要ないです。

carried_textures

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "brightness_gamma": 0, "carried_textures": "cobblestone", "isotropic": false, "sound": "stone", "textures": "cobblestone" }
}

　インベントリ内でのテクスチャを置いた時のテクスチャと別にするときに指定します。

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "brightness_gamma": 0,
"carried_textures": { "up": "cobblestone", "down": "cobblestone", "north": "cobblestone", "south": "cobblestone", "east": "cobblestone", "west": "cobblestone" },
"isotropic": false, "sound": "stone", "textures": "cobblestone" }
}

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "brightness_gamma": 0,
"carried_textures": { "up": "cobblestone", "down": "cobblestone", "side": "cobblestone" },
"isotropic": false, "sound": "stone", "textures": "cobblestone" }
}

　これもまた、各面に対して指定できます。

isotropic

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "brightness_gamma": 0, "carried_textures": "cobblestone", "isotropic": false, "sound": "stone", "textures": "cobblestone" }
}

　テクスチャの向きがブロックの設置場所によって0°, 90°, 180°, 270°とランダムに変わるようにするかどうか。

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "brightness_gamma": 0, "carried_textures": "cobblestone",
"isotropic": { "up": false, "down": false, "north": false, "south": false, "east": false, "west": false },
"sound": "stone", "textures": "cobblestone" }
}

{ "format_version": [ 1, 1, 0 ],
"minecraft:cobblestone": { "brightness_gamma": 0, "carried_textures": "cobblestone",
"isotropic": { "up": false, "down": false, "side": false },
"sound": "stone", "textures": "cobblestone" }
}

　これもまた、各面に対して指定できます。
