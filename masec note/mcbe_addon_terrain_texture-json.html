アドオン作成 - terrain_textures.jsonについて

　このファイルはブロックのテクスチャに関するファイルで、リソースパックのtexturesフォルダに一つだけあります。

━ ┳ manifest.json
　 ┣ pack_icon.png
　 ┗ textures
　 　 ┗ terrain_textures.json
機能

　ブロックにどんなテクスチャを貼るかを決定するための中間行程です。
　主な役割として、テクスチャとTerrainIDを結びつけます。

ブロック本体
　↓　block.jsonで紐づけ
TerrainID
　↓　terrain_texture.jsonで紐づけ
テクスチャ
ファイル内容

　本来は、テクスチャを紐付けしたいブロックすべてについて書きますが、はじめは例として一つのブロックの場合を示します。

Res
textures/terrain_textures.json
{
  "num_mip_levels": 4,
  "padding": 8,
  "resource_pack_name": "vanilla",
  "texture_name": "atlas.terrain",
  "texture_data": {
    "cobblestone": { "textures": "textures/blocks/cobblestone" }
  }
}

　概形はこんな感じです。
　自分で新しく作る場合はこれ↑をコピペするといいでしょう。

{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": {
"cobblestone": { "textures": "textures/blocks/cobblestone" }
} }

　この赤線で囲った部分が一つのカタマリです。これを適当に「ユニット」と呼ぶことにします。
(このサイト独自の呼び方です)

{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": { "cobblestone": { "textures": "textures/blocks/cobblestone" } } }

　上のこの部分に関しては、変えても特におもしろいことは起きないので説明を省きます。コピペしたらそのままでいいと思います。

TerrainID

{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": {
"cobblestone": { "textures": "textures/blocks/cobblestone" }
} }

　便宜上、この部分をTerrainIDと呼びます。(このサイト独自の呼び方です)
　そしてこれは各自で自由に決められます。

テクスチャを指定

{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": {
"cobblestone": { "textures": "textures/blocks/cobblestone" }
} }

　ここにテクスチャのファイルの場所(ファイルパス)を指定します。

指定するブロックを増やす

{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": {
"cobblestone": { "textures": "textures/blocks/cobblestone" },
"dirt": { "textures": "textures/blocks/dirt" }
} }

　複数のブロックについて記述する場合は、ユニット自体を増やします。
　ユニットとユニットをつなぐときだけカンマ( , )を書きます。忘れるとエラーになります。

ブロックにテクスチャがつくまで

　普通に6面同じテクスチャなら単純です。textures/blocksにある「abc.png」を「test:my_block」というidのブロックに貼るとします。
　そして、TerrainIDも自分で決められるのでyamaimoとします。すると、

Res
textures/terrain_textures.json
{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": {
"yamaimo": { "textures": "textures/blocks/abc" }
} }

Res
blocks.json
{ "format_version": [ 1, 1, 0 ],
"test:my_block": { "sound": "stone", "textures": "yamaimo" }
}

　このようにTerrainIDを介してテクスチャとブロック本体が結びつきます。
　一方、面で異なる場合は慣れていないと一見複雑に感じます。例として、上面、側面、下面で異なるテクスチャ「abc.png」「def.png」「ghi.png」であるとしましょう。この場合、

Res
textures/terrain_textures.json
{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": {
"yamaimo_up": { "textures": "textures/blocks/abc" },
"yamaimo_side": { "textures": "textures/blocks/def" },
"yamaimo_down": { "textures": "textures/blocks/ghi" }
} }

Res
blocks.json
{ "format_version": [ 1, 1, 0 ],
"test:my_block": { "sound": "stone", "textures": { "up": "yamaimo_up", "side": "yamaimo_side", "down": "yamaimo_down" } }
}

　というようにTerrainIDを分岐させて指定します。

発展的な内容

複数のテクスチャ指定

{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": {
"anvil_top_damaged_x": { "textures": [ "textures/blocks/anvil_top_damaged_0", "textures/blocks/anvil_top_damaged_1", "textures/blocks/anvil_top_damaged_2", "textures/blocks/anvil_base" ] }
} }

　もともとデータ値をもっているブロックならば、上から順にデータ値0,1,2,...のテクスチャになります。
　また、別のところで何番目か選択できる場合はこのように書いて選択肢を持たせます。

overlay_color

{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": {
"grass_carried": { "textures": { "overlay_color" : "#79c05a", "path" : "textures/blocks/grass_side" } }
} }

　指定した単色を重ねることができます。16進カラーコードで指定します。

variations

{ "num_mip_levels": 4, "padding": 8, "resource_pack_name": "vanilla", "texture_name": "atlas.terrain", "texture_data": {
"bookshelf": { "textures": { "variations" : [ { "path": "textures/blocks/bookshelf0" }, { "path": "textures/blocks/bookshelf1" }, { "path": "textures/blocks/bookshelf2" }, { "path": "textures/blocks/bookshelf3" } ] } }
} }

　ブロックの座標ごとにランダムなテクスチャにすることができます。
　余談ですが、バニラでこの書き方は使われていません。
