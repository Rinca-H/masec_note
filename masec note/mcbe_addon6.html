アドオン作成 - アイテムの作り方

目次は右上の「≡」から！

　オリジナルのアイテムの作り方・追加方法を紹介します。 オリジナルのものを「追加する」ことになるので、ある程度アドオンに慣れていないとつまずくかもしれません。
　複雑なものほど急に難しくなるので注意してください。

基本的な流れ

　もともとマイクラになかったものを追加するので、アイテム本体に加えてテクスチャも追加する必要があります。 つまり、ビヘイビアとリソース両方が必要になります。
　全体の作業はこんな感じです：

ビヘイビア
・itemsフォルダに新しくJSONを追加する
リソース
・texturesフォルダのitem_texture.jsonを編集・作成する
・アイテムのテクスチャを追加する

アイテム本体を定義

　まず、ビヘイビアパック内にitemsフォルダを用意します。

━ ┳ manifest.json
　 ┣ items
　 ┗ pack_icon.png

　そのフォルダのなかにアイテム本体を定義するJSONを、追加するアイテムの数だけ作って入れることになります。

テクスチャを用意

　自分で場所を指定することができますが、オススメは「(リソースパック)/textures/items」の中です。

━ ┳ manifest.json
　 ┣ pack_icon.png
　 ┗ textures
　 　 ┗items
　 　 　 ┗〇〇.png

テクスチャと本体を紐づけ

「(リソースパック)/textures」の中に「item_texture.json」を用意します。
　元からあれば編集、なければ新しく作ります。
　詳しくはこちら → item_texture.jsonについて

アイテム本体(基礎)

　これがまだ何も指定していない状態のアイテムです。ゼロから作る場合はこれをコピペするといいでしょう。

Dat
items/my_item.json
{
  "format_version": "1.16.100",
  "minecraft:item": {
    "description": {
      "identifier": "test:my_item",
      "category": "Items"
    },
    "components": {
      "minecraft:icon": {
        "texture": "my_item"
      }
      // コンポーネント
    }
  }
}

主な構成

{ "format_version": "1.16.100", "minecraft:item": { "description": { "identifier": "test:my_item", "category": "Items" },
"components": { "minecraft:icon": { "texture": "my_item" } // コンポーネント }
} }

　大きく関わってくるのがこの赤い枠で囲った部分で、もっと言えばその中の{ }内にコンポーネントを書き足していきます。
　minecraft:iconもコンポーネントですが、テクスチャを指定するために必須なので書いておきました。

IDを決める

{ "format_version": "1.16.100", "minecraft:item": { "description": { "identifier": "test:my_item", "category": "Items" },
"components": { "minecraft:icon": { "texture": "my_item" } // コンポーネント }
} }

　IDを決め忘れないでください。

？
identifier(ID)の決め方
　基本的には「(アドオンid):(本id)」といった感じ。
　アドオンidはアドオン全体で統一する。苗字のようなもの。ネームスペースとかプリフィックスとか呼ばれることがある。
　本idは文字通り本体のid。みんな違うものをつけましょう。
例)　test:my_content
--注意--
・どちらも先頭を数字にしない。
・数字かアルファベットのみを使う。
・アドオンidに「minecraft」は使ってはならない。
・アドオンidは、ほかのアドオンとの競合を考えてオリジナル性の高いものにする。

TerranIDを指定する

{ "format_version": "1.16.100", "minecraft:item": { "description": { "identifier": "test:my_item", "category": "Items" },
"components": { "minecraft:icon": { "texture": "my_item" } // コンポーネント }
} }

　ここでTerrainIDを指定します。

コンポーネントを追加する

{ "format_version": "1.16.100", "minecraft:item": { "description": { "identifier": "test:my_item", "category": "Items" },
"components": { "minecraft:icon": { "texture": "my_item" }, "minecraft:allow_off_hand": true }
} }

　リファレンスから選んでコンポーネントを追加します。追加する際はカンマに気を付けてください。忘れるとエラーになります。
　何を選んだらいいのかわからない場合は後で例をいくつか載せるので参考にしてください。

コンポーネントの選び方

　Items.htmlというリファレンスの「Item Components」の章に載っているものはすべてコンポーネントです。
　しかし、選んでもコンポーネントの形がリファレンスからは分かりづらいのでこちらを参考にしてみてください。

最も簡単な例

　JSONがシンプルになるように性質を選んできています。
　しかし、もともと簡単なコンポーネントが少ないのでスカスカです

Dat
items/simple_item.json
{
  "format_version": "1.16.100",
  "minecraft:block": {
    "description": {
      "identifier": "test:simple_item"
    },
    "components": {
      "minecraft:fuel": {
        "duration": 80
      },
      "minecraft:icon": {
        "texture": "simple_item"
      },
      "minecraft:max_stack_size": 16
    }
  }
}

　このお手本アイテムは以下のような性質を持ちます↓

・かまどでの燃焼時間が80秒(アイテム8個分、石炭と同じ)
・TerrainIDはsimple_itemである
・16個までしかスタックできない

　はじめはこれをコピペして自分で値を変えて試すのがいいかもしれません。
　それぞれの性質についてもっと詳しく説明します。

minecraft:fuel

　かまどの燃料として使ったときに火が付いている秒数。 指定した秒数を10で割った時の数が、この燃料一つ分で

minecraft:max_stack_size

　スタック数の上限。最大は99です。
